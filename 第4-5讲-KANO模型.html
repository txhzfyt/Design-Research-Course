KANO模型

课程定位

通过KANO分类用户需求层次，指导功能优先级与资源分配，提升满意度与产品质量决策的科学性。

学习目标

  • 理解KANO五类需求与心理机制
  • 掌握问卷设计与分类判定流程
  • 应用Better-Worse系数与矩阵优先级
  • 完成案例的需求映射与界面转化

KANO概念与需求类别

概念

KANO模型由狩野纪昭提出，用于从用户心理认知角度分类需求并排序，建立产品表现与满意度的结构化关系。

五类需求

  • M 基本型：缺失极度不满，满足提升有限
  • O 期望型：线性提升，越好越满意
  • A 魅力型：超预期带来惊喜，不有亦无不满
  • I 无差异：提供与否满意度影响不显著
  • R 反向型：提供反而降低满意度

实施流程与问卷设计

实施步骤

流程要点

  • 需求收集 → 问卷设计（正/反提问） → 数据分析
  • 分类判定：KANO评价表与Better-Worse系数
  • 矩阵绘制与决策：先M，再O，后A

正反提问与量表

问卷设计原则

  • 每个需求设置两问：功能存在/功能缺失
  • 采用5级量表：喜欢、理所当然、无所谓、勉强接受、不喜欢
  • 针对社交、搜索、规划、优化、推送等通用功能设计题项

Better-Worse系数与说明

计算公式

  • Better：(A + O) / (A + O + M + I)
  • Worse：(O + M) / (A + O + M + I)

用于量化需求提升满意度的潜力与减少不满的潜力，辅助优先级排序与资源配置。

四象限优先级

Better-Worse散点图

优先策略

  • 第四象限M优先满足，保障基本体验
  • 第一象限O优化投入，形成竞争力
  • 第二象限A适度创新，提升忠诚度

案例总述：智能行程助手APP

场景与目标

服务旅游列车出行的碎片化体验，打通行程规划、路线选择、信息推送与社交分享的全链路服务。

方法框架

  • 访谈/调研 → 场景化需求 → KANO分类 → 优先级
  • 结合Better-Worse系数，绘制矩阵并制定策略
